Behaviors

5 nodes in this area of the perspective

This hub gathers the system behaviors of a git-native memory product — both the low-level operations and the user-facing workflow. The canonical operation set is enumerated as [Compile/Extract/Diff/TX/Commit/Evolve] (Narrative_SIC_Behaviors_1), the semantic-compilation primitives. At the workflow layer these sequence into a [create-memory → posit → remember chain] (Obs_WorkflowBehaviors), where posit previews and remember commits. The framing throughout is [intentional rather than automatic memory] (Observation_IntentionalMemory): an agent chooses what to remember, offers it to the user, then saves it to git — explicitly opposed to context autosummarization. This sits inside the broader [git-native collaborative worldview] (Narrative_GitNativeMemory) that differentiates the product from model-held built-in memory (intentional memory, multiplayer AI, progressive compilation). the customer's chief architect also flagged a gap the behavior set does not yet cover: an [onboarding mode] (Feature_OnboardingMode) to bootstrap collective memory from knowledge in people's heads rather than from ongoing AI work.
