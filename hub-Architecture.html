Architecture

7 nodes in this area of the perspective

The Architecture hub frames an organization's worldview as the single source of truth from which everything else is rendered. The mainline is a compilation model: [worldview as source, outputs as compiled targets] (Narrative_CompilationTargets) treats content, apps, and code as renders from worldview state. This rests on a [Git-centric repo of record] (Narrative_RepoAsSourceOfTruth) — content committed before extraction, GitHub Actions for async processing, append-only transactions — and is made legible by [branchable worldview] (Narrative_BranchingWorldview), where direction changes are explicit, reviewable, testable like code. Two feedback/interface mechanisms sit alongside: [telltales as drift detectors] (Narrative_Telltales), recompiled stories that surface worldview drift, and a [dual chat/Discourse interface] (Narrative_TwoInterfaceModel) splitting fast iteration from async collaboration. The motivating premise is the [white-space argument] (Narrative_WhiteSpace): absent an explicit worldview, AI agents default to their training distribution, so worldview engineering prevents drift — and the payoff named is [agents that code with business strategy] (ValueProp_4).
